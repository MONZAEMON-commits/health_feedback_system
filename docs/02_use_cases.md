# 2. 利用シーン（Use Cases）

本章では、本システムを利用するユーザーが想定される操作の流れを整理し、  
利用状況に応じたユースケースを明確にする。

本システムは「一般社員」「管理者レベル1」「管理者レベル2」の 3 区分で利用され、  
それぞれの目的や権限に応じて異なる操作が提供される。

---

## 2.1 一般社員の利用シーン

### ● 1. ログイン
- 出社時にログインし、当日の体調評価を入力する。
- ログイン方式は Django 標準の認証を使用（ICカード認証は将来的拡張）。
- 10秒以上操作しない場合は自動ログアウト。

### ● 2. 体調・気分の入力
- 肉体的・精神的それぞれ 1〜5 のアイコンをタップして入力。
- 1日1回入力が原則（上書き不可）。
- 画面幅に応じてアイコンが 1〜2段に自動切り替え。（レスポンシブ対応）

### ● 3. 入力確認
- “登録” ボタンを押すと、内容を確認するテキストボックスが表示される。
- 再確認後に確定ボタンを押すと保存される。

### ● 4. 入力完了画面
- 登録内容を簡易表示。
- 「ログアウト」ボタンを促す。
- 操作がない場合は 10 秒後に自動ログアウト。

### ● 5. ご意見箱への任意投稿
- 体調入力画面の端にアイコンを設置し、任意でご意見箱へ遷移できる。
- 匿名／準匿名／署名の 3 種類から選んで投稿可能。

---

## 2.2 管理者（レベル1）の利用シーン

レベル1管理者は氏名を含む個人データの閲覧が可能。

### ● 1. ログイン
- 権限レベルに応じて管理者画面の内容が切り替わる。

### ● 2. 統計表示（個人／部署／全体）
- 体調・気分の平均値表示
- 部署別一覧（氏名表示あり）
- 欠勤は NULL として扱い、集計対象外

### ● 3. フォロー対象の把握
- 一定期間（例：1週間）で低評価が続く社員を確認
- 必要に応じて声掛けなどのフォローが可能

### ● 4. ご意見内容の閲覧（準匿名／署名のみ）
- 匿名投稿は閲覧不可
- CSVを読み込み、対象者の意図を確認可能

### ● 5. 管理者画面の自動ログアウト
- 無操作の状態が指定した時間（1〜60分）を超えると自動ログアウト

---

## 2.3 管理者（レベル2）の利用シーン

レベル2は匿名性確保のため、氏名・個人情報の閲覧はできない。

### ● 1. ログイン
- レベル2権限に応じて匿名化された画面が表示される。

### ● 2. 匿名化された統計の閲覧
- 氏名情報なし（社員番号も非表示）
- 部署単位・全体単位の傾向のみ表示

### ● 3. 匿名／準匿名／署名投稿の統計閲覧
- 投稿内容はテキストのみ確認可能
- 誰が投稿したかの情報は閲覧不可

### ● 4. 管理者画面の自動ログアウト
- 無操作のまま指定時間（1〜60分）を超えると強制ログアウト

---

## 2.4 システム全体の利用シーケンス（概要）

```text
一般社員：
1. ログイン  
2. 体調入力  
3. 確認  
4. 登録  
5. 完了画面（自動ログアウト）

管理者：
1. ログイン  
2. 統計確認（レベルに応じた表示）  
3. 必要に応じてフォロー検討  
4. 一定時間後自動ログアウト
```
---

## 2.5 特記事項（例外シナリオ）

- **欠勤時（出社しない場合）**  
  入力は行わず、当日のレコードは NULL として管理する。  
  → 集計から自動で除外される。

- **中断された入力**  
  未確定の場合はデータは保存されない。  
  → 再ログインで再入力可能。

- **機器操作が不慣れな社員のフォロー**  
  簡易 UI とし、操作エラーが起きないように画面設計する。
