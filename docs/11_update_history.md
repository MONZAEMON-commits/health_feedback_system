## 11.2 今後の更新方針

- 仕様修正や追加があれば **v1.1, v1.2…** と increment  
- フロー追加・画面追加・DB修正などの大きい変更は minor version up  
- デザイン調整や文章微修正は patch 扱い（v1.0.1 など）でも良い
