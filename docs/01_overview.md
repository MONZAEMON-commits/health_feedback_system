# 1. 概要（Overview）

本システムは、**社員の体調管理** と **ご意見箱投稿** の 2 種類の機能を提供する Web アプリケーションである。  
日次の体調記録によって社員の心身状態の傾向を可視化するとともに、匿名性を確保した意見投稿機能によって職場改善に役立てることを目的とする。

主要技術として、体調管理システムは **Django（Python）** を用いて構築し、  
ご意見箱は **PHP** で実装する。両者はデータベースを共有せず、**CSV を介した読み取り専用の連携**により分析を行う。  

また、体調管理の入力画面からご意見箱への導線を設けることで、社員が日常的に利用できるシンプルな UI を目指す。

---

## 1.1 目的（Purpose）

- 社員の体調（肉体・精神）を日々蓄積し、長期的な傾向を可視化する  
- 管理者が組織全体や部署単位の統計を把握できるようにする  
- 匿名／準匿名／署名による意見投稿を可能にし、心理的安全性を高める  
- 投稿データと体調データを用いて、将来的な分析拡張の基盤をつくる  
- ローカル環境で実行できる、個人開発向けの軽量なシステムを構築する  

---

## 1.2 システム構成（System Components）

本システムは、以下の 2 モジュールで構成される。

### ① 体調管理モジュール（Django）

- 日次の体調（肉体／精神）入力  
- 管理者画面での統計グラフ表示  
- pandas による簡易分析  
- 欠勤時は体調データを NULL として扱い、集計対象外とする  

### ② ご意見箱モジュール（PHP）

- 匿名／準匿名／署名の 3 モードで投稿可能  
- 投稿内容は CSV に保存  
- Django 側が必要に応じて CSV を読み込み分析する  

---

## 1.3 利用形態（Usage）

- **ローカル環境で利用可能**  
- Windows + Django + PHP で動作  
- SQLite と CSV でデータを管理  
- スマホからアクセス可能なレスポンシブ構成  
- サーバー構築不要（ローカルで完結）

管理者は 2 段階のレベルを使用する。

| 権限レベル    | 内容                                                                 |
| ------------- | -------------------------------------------------------------------- |
| ------------- | -------------------------------------------------------------------- |
| 管理者レベル1 | 個人と紐づく体調データの集計を閲覧可能（匿名投稿の内容閲覧は不可）   |
| 管理者レベル2 | 匿名化された集計を閲覧可能／匿名投稿の内容・タグを閲覧可能           |
